We have a clearer direction for AIH:

AI Hallucination testing as a practical evaluation framework for agentic AI.

The idea is simple:

1. Define bounded AIH tests.

2. Modify HybridAI or another agentic AI stack to execute them.

3. Run the tests.

4. Document the results and statistics.

5. Use the results to guide the next implementation.

The current AIH structure has three test families:

AIhistory:

tests based on real agentic-AI failures, including provenance hallucination,
continuity errors, unstable decisions, and project-flow disturbance.

k-phd:

a knowledge-ladder test path from K-12 through college, graduate, and PhD-level
work, with limits on time, I/O, grading, and reference sources.

AIchess:

a bounded chess-match test for state fidelity, legal moves, timing, move
faults, and agent reliability.

The point is not to claim that one AI stack is generally "better." The point
is to compare HybridAI, Codex, cloud agents, local models, and future agentic
systems under the same bounded tests.

That lets us do two things:

1-plan intelligently,

2-demonstrate capability through observed results.
